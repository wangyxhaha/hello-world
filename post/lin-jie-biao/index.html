在做图论的时候，为了储存图，我们会使用邻接矩阵

就比如上面这个图，用邻接矩阵储存就是这样：

0 1 2 3 4 5
0 0 0 1 0 1 1
1 0 0 0 0 1 1
2 1 0 0 1 1 0
3 0 0 1 0 0 0
4 1 1 1 0 0 0
5 1 1 0 0 0 0

但是这样储存明显有缺点：

  1. 空间复杂度太大 O(n2)O(n^2)O(n2)
  2. 时间复杂度太大 O(n2)O(n^2)O(n2)

于是乎，邻接表你值得拥有

首先我想说邻接表真是个巧妙的数据结构呵

邻接表

首先，邻接表跟链表有几分相像，这样就好理解邻接表了

不过我们不用链表，我们只是用数组来模拟邻接表的普通OIer

那么就有以下数组要用：

  1. u[i]u[i]u[i] 存第iii条线的起点
  2. v[i]v[i]v[i] 存第iii条线的终点
  3. w[i]w[i]w[i] 存第iii条线的权值
  4. first[i]first[i]first[i] 存我其实说不清楚
  5. next[i]next[i]next[i] 存其实我还是说不清楚

还有，设nnn为边数，mmm为点数

关于uuu,vvv,www数组

读入的时候，就把对应的起点、终点、权值存进uuu vvv www

关于firstfirstfirst,nextnextnext数组

最重要的地方来了，就是firstfirstfirst和nextnextnext到底是干啥的呢？

关于firstfirstfirst

firstfirstfirst数组其实是存起点为第iii个点的边的编号

拗口？（反正我这么觉得）

实际操作起来就是这样的：

初始化将firstfirstfirst全部初始化为−1-1−1

每输入一条边（假设输入的边的编号、起点、终点、权值分别为iii,uiu_iui​,viv_ivi​,wiw_iwi​），first[ui]first[u_i]first[ui​]若等于−1-1−1，就将first[ui]first[u_i]first[ui​]赋为iii

现在根据这个图（权值就看做1）

uuu vvv www可以是这样的（反正什么顺序都一样，就字典序吧）：

iii 1 2 3 4 5 6 7 8
uuu 0 0 0 1 1 2 2 4
vvv 2 4 5 4 5 3 4 5
www 1 1 1 1 1 1 1 1

然后，拿出firstfirstfirst数组，全部初始化为−1-1−1

jjj 0 1 2 3 4 5
firstfirstfirst -1 -1 -1 -1 -1 -1

按照步骤，读到i=1i=1i=1的时候firstfirstfirst数组就是这样：

jjj 0 1 2 3 4 5
firstfirstfirst 1 -1 -1 -1 -1 -1

可以说，邻接表的操作方式真的很迷惑

但接下来就是更迷惑的nextnextnext数组了

关于nextnextnext

当有两条边的起点一样时（设第一个边编号为iii，第二个边编号为jjj，公共的起点为xxx，就是说 ui=uj=xu_i=u_j=xui​=uj​=x ），nextnextnext就发挥了作用

如果我们在这种情况下仍然用上面firstfirstfirst的操作的话，以前的first[x]first[x]first[x]自然会被覆盖掉

所以我们把first[x]first[x]first[x]存到next[j]next[j]next[j]去，再把first[x]first[x]first[x]赋为jjj

拗口？（反正我还是这么觉得）

实际上就是这样：

初始化时nextnextnext数组全部初始化为−1-1−1

每次在输入一条边后，如果first[x]first[x]first[x]不等于−1-1−1，那就将next[j]next[j]next[j]赋为first[x]first[x]first[x]，再将first[x]first[x]first[x]赋为jjj

还是接着之前的操作：

jjj 0 1 2 3 4 5
firstfirstfirst 1 -1 -1 -1 -1 -1
kkk 1 2 3 4 5 6 7 8
nextnextnext -1 -1 -1 -1 -1 -1 -1 -1

按步骤操作到i=2i=2i=2

jjj 0 1 2 3 4 5
firstfirstfirst 2 -1 -1 -1 -1 -1
kkk 1 2 3 4 5 6 7 8
nextnextnext -1 1 -1 -1 -1 -1 -1 -1

真的非常迷惑

好了，我们现在存好数组了，那么如何读取呢？

这个费尽心思搞出的储存方式，一定有很好的性能（简洁的算法往往带来低效，复杂的算法往往带来高效）

读取

我们来看，当我们需要遍历以一个点为起点的所有边，我们怎样来读取信息

以000为起点为样例：（为了演示就只能用图片了）

黄的表示起点，蓝的表示边的编号，红的表示下一条边的储存位置，黑的表示读取终止

其中我们从firstfirstfirst数组的第000项读起

下一个边的位置便是当前读到的边的编号

直到读到−1-1−1为止

你会发现，读取的效率非常高，因为读取都是一连串读下来，次次读的都是数据

而且空间占用很小

对比一下邻接矩阵，邻接矩阵无地自容
